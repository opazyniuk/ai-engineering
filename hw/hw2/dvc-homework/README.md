# DVC Homework — Data Versioning

## Завдання

У вас є брудний датасет `data/dataset_v1.csv`. Потрібно:

1. Створити git repo з DVC в папці `dvc_workspace/`
2. Додати датасет під DVC tracking і закомітити як **v1** (брудні дані)
3. Почистити дані і закомітити як **v2** (чисті дані)
4. Показати що можна переключатись між v1 і v2

Як саме організувати код — вирішуєте самі. Можна один скрипт, можна декілька, можна вручну в терміналі.

---

## Що не так з даними

Відкрийте `data/dataset_v1.csv` і знайдіть проблеми:

- **Дублікати** — деякі рядки повторюються (однаковий id)
- **Пропущені значення** — є порожні клітинки (value або date або category)
- **Неконсистентні labels** — "ENTERPRISE" vs "enterprise" (різний регістр)
- **Неправильний label** — Bob має category "smb", а має бути "enterprise"
- **Неправильне значення** — Hank має value 4200, а має бути 4800

---

## Вимоги до v2 (чистий датасет)

Після очистки `dataset.csv` має:
- Не мати дублікатів
- Не мати пропущених значень (рядки з null — видалити)
- Всі category в lowercase
- Bob = enterprise
- Hank = 4800
- Мати рівно 10 рядків даних (+ header)
